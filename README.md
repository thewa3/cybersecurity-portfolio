# 🛡️ Home Lab Setup — Cybersecurity Practice Environment

## Overview
This project documents the setup of a personal cybersecurity home lab built using VirtualBox/VMware. The lab was created to practice penetration testing, network analysis, and ethical hacking techniques in a safe, controlled environment.

---

## Environment

| Component | Purpose |
|-----------|---------|
| **Kali Linux** | Attacker machine — main pentesting OS with built-in security tools |
| **Windows 7** | Target machine — vulnerable OS used for exploitation practice |
| **Windows Server** | Target server environment — simulates real-world corporate networks |
| **Firewall** | Network security control — used to practice traffic filtering and rule configuration |
| **Metasploit Framework** | Exploitation framework — used to launch and test attacks against target machines |

---

## What I Learned

- How to install and configure multiple virtual machines on one physical computer
- How to set up an isolated network so lab traffic does not affect real networks
- How to connect attacker and target machines in a controlled environment
- The difference between attacker machines (Kali Linux) and target machines (Windows)
- Basic network configuration including IP addressing and firewall rules

---

## Tools Used

- VirtualBox / VMware
- Kali Linux
- Metasploit Framework
- Windows 7
- Windows Server

---

## Why This Matters

Setting up a home lab is the foundation of practical cybersecurity. It allows ethical hackers to practice real attack and defense techniques without causing harm to real systems. This lab environment was used for all subsequent projects in this portfolio.
